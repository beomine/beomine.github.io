## 폭발적으로 성장한 에이전트 생태계

2026년, Claude Code를 중심으로 **에이전트 & 스킬 오픈소스 생태계**가 폭발적으로 성장했다. 6개월 전만 해도 `CLAUDE.md` 하나 작성하면 충분했지만, 지금은 수백~수천 개의 에이전트와 스킬을 묶어 배포하는 프로젝트가 여럿 등장했다.

문제는 **선택지가 너무 많다**는 것이다. 이 글에서는 주요 프로젝트 6개를 비교 분석하고, 상황별 선택 가이드를 제공한다.

## 1. wshobson/agents — 가장 인기 있는 올인원 시스템

> GitHub Stars: **32.7k** / 가장 활발한 커뮤니티

### 개요

Claude Code를 위한 **프로덕션 레디 멀티 에이전트 시스템**이다. 단순한 에이전트 모음이 아니라, 에이전트 간 협업(오케스트레이션)까지 설계되어 있다.

### 구성

| 카테고리 | 수량 | 설명 |
|----------|:---:|------|
| 에이전트 | 182개 | 아키텍처, 언어, 인프라, 품질, AI, 문서, SEO 등 |
| 스킬 | 147개 | 모듈형 지식 패키지, 점진적 공개 방식 |
| 플러그인 | 75개 | 단일 목적, 최소 토큰 사용 최적화 |
| 오케스트레이터 | 16개 | 멀티 에이전트 워크플로우 조율 |

### 특징: 팀 기반 오케스트레이션

여러 에이전트를 **팀으로 묶어 병렬 실행**할 수 있다는 점이 최대 강점이다.

| 프리셋 팀 | 구성 |
|-----------|------|
| review | 코드 리뷰 에이전트 팀 |
| debug | 가설 기반 디버깅 팀 |
| feature | 기능 개발 팀 |
| fullstack | 풀스택 개발 팀 |
| security | 보안 감사 팀 |
| migration | 마이그레이션 팀 |

### 사용법

자연어로 요청하면 적절한 에이전트가 자동 선택되고, 슬래시 명령어로 직접 호출도 가능하다.

### 이런 사람에게 추천

- Claude Code를 **메인 개발 도구**로 쓰는 개발자
- **멀티 에이전트 협업**이 필요한 복잡한 프로젝트
- 가장 활발한 커뮤니티와 업데이트를 원하는 경우

## 2. Antigravity Awesome Skills — 최대 규모

> 1,370+ 스킬 / 6개 도구 지원

### 개요

현존하는 **가장 큰 스킬 라이브러리**다. 계획, 코딩, 디버깅, 테스트, 보안, 인프라, 제품, 성장 등 거의 모든 영역을 커버한다.

### 구성

- **1,370+ 스킬** — 역할 기반 번들(프론트엔드, 백엔드, 인프라, 보안, 제품)
- **멀티스텝 워크플로우** — 계획 → 구현 → 테스트 → 배포 가이드
- **플러그인 배포** — Claude Code 마켓플레이스 지원

### 설치

```bash
# 전체 설치
npx antigravity-awesome-skills

# Claude Code 전용
npx antigravity-awesome-skills --claude

# 마켓플레이스 플러그인
/plugin marketplace add sickn33/antigravity-awesome-skills
```

### 주의사항

전체 설치 시 **첫 로딩에 10~15분**이 걸릴 수 있다. 에이전트가 모든 SKILL.md 메타데이터를 읽고 인덱싱해야 하기 때문이다. 필요한 번들만 선택 설치하는 것을 권장한다.

### 이런 사람에게 추천

- **여러 AI 코딩 도구**를 동시에 사용하는 경우 (Claude Code + Cursor 등)
- 가능한 한 **많은 영역을 커버**하고 싶은 경우
- 번들 단위로 필요한 것만 골라 쓰고 싶은 경우

## 3. claude-skills — Python 개발자의 선택

> 5.2k Stars / 223 스킬 + 332 Python CLI 도구

### 개요

스킬뿐 아니라 **실행 가능한 Python CLI 도구**가 함께 제공된다는 점이 독보적이다. 개발 외에도 마케팅, 재무, 컴플라이언스 등 비개발 영역까지 커버한다.

### 구성

| 카테고리 | 수량 |
|----------|:---:|
| 스킬 | 223개 |
| Python CLI 도구 | 332개 |
| 레퍼런스 가이드 | 416개 |
| 에이전트 | 23개 |
| 슬래시 명령어 | 22개 |

### 특징: 제로 의존성 Python 도구

332개의 Python 도구가 **표준 라이브러리만** 사용한다. `pip install` 없이 Python만 있으면 바로 실행 가능하다.

### 도메인

```
engineering / product / marketing / compliance
C-level advisory / project management / business growth / finance
```

### 설치

```bash
/plugin marketplace add alirezarezvani/claude-skills
/plugin install <name>@claude-code-skills
```

### 이런 사람에게 추천

- **Python 중심** 개발자
- 개발 외 **비즈니스 영역**(마케팅, 재무 등)에도 AI를 활용하고 싶은 경우
- 실행 가능한 CLI 도구가 필요한 경우

## 4. agency-agents — AI 에이전시 시뮬레이션

> 193 에이전트 / 14개 도구 지원

### 개요

AI 에이전시의 **팀원 역할**을 시뮬레이션하는 프로젝트다. 각 에이전트에 성격, 미션, 프로세스, 성공 지표가 정의되어 있다.

### 특징: 페르소나 기반 에이전트

다른 프로젝트가 "무엇을 하는지"에 초점을 맞춘다면, Agency Agents는 **"누가 하는지"**에 초점을 맞춘다.

| 역할 예시 | 설명 |
|-----------|------|
| Frontend Wizard | 프론트엔드 전문가, 접근성 우선 |
| Reddit Community Ninja | 커뮤니티 성장 전문가 |
| Whimsy Injector | 마이크로 인터랙션, 재미 요소 전문 |
| Reality Checker | 현실성 검증, 기술 부채 경고 |

### 최대 도구 지원

**14개 도구**를 지원하는 것이 가장 큰 장점이다. Claude Code는 물론 Cursor, Aider 등 어떤 도구를 쓰든 동일한 에이전트를 사용할 수 있다.

### 설치 (Claude Code)

에이전트 파일을 `~/.claude/agents/`에 복사하면 바로 사용 가능하다. 마크다운 + YAML frontmatter 형식이라 변환이 필요 없다.

### 이런 사람에게 추천

- **여러 AI 도구를 혼용**하면서 일관된 에이전트를 쓰고 싶은 경우
- 개발 외에 **마케팅, 디자인, 커뮤니티** 영역에도 에이전트를 활용하고 싶은 경우
- 에이전트에 **캐릭터와 맥락**을 부여하고 싶은 경우

## 5. ECC (Everything Claude Code) — 하네스 프레임워크

> 이전 시리즈에서 자세히 다룸: [ECC 프로젝트 분석](/posts/Everything-Claude-Code-프로젝트-분석/)

### 개요

에이전트나 스킬 컬렉션이 아니라, Claude Code의 **전체 동작을 제어하는 하네스 시스템**이다.

### 다른 프로젝트와의 차이

| | 에이전트/스킬 프로젝트 | ECC |
|---|---|---|
| 제공하는 것 | "이런 전문가가 있다" | "이런 규칙으로 일한다" |
| 초점 | 역할, 기능 | 프로세스, 품질, 자동화 |
| 구성 | agents, skills | rules, hooks, agents, skills, memory |

### 이런 사람에게 추천

- **코드 품질과 프로세스 일관성**이 가장 중요한 경우
- 자동 포맷팅, 린트, 보안 검사 등 **자동화**를 원하는 경우
- 하네스 시스템의 **기반을 먼저 갖추고** 싶은 경우

## 6. awesome-claude-code — 리소스 큐레이션

> 직접 에이전트를 제공하지 않고, 좋은 프로젝트를 **모아서 소개**하는 큐레이션 리스트

유용한 스킬, 에이전트, 플러그인, MCP 서버 등을 한곳에서 찾을 수 있다. 뭘 써야 할지 모를 때 여기서 시작하면 좋다.

## 상황별 선택 가이드

### "하나만 고르라면?"

**wshobson/agents** — 가장 큰 커뮤니티, 가장 활발한 업데이트, 오케스트레이션까지 지원. Claude Code만 쓴다면 이것부터 시작.

### 목적별 추천

| 상황 | 추천 |
|------|------|
| Claude Code 올인, 복잡한 프로젝트 | **wshobson/agents** |
| 여러 도구 혼용 (Cursor + Claude Code) | **Antigravity** 또는 **agency-agents** |
| Python 중심 + 비즈니스 영역 확장 | **claude-skills** |
| 코드 품질/프로세스 자동화 우선 | **ECC** |
| 전체 생태계 탐색 | **awesome-claude-code** |

### 조합 추천

이들은 경쟁이 아닌 **보완 관계**다. 추천 조합:

```
기반: ECC (하네스 시스템)
  +
에이전트: wshobson/agents (개발 에이전트)
  또는
에이전트: agency-agents (멀티 도구)
```

ECC로 규칙과 자동화를 깔고, 그 위에 에이전트 컬렉션을 올리는 구조가 가장 효과적이다.

## 주의할 점

### 1. 전부 설치하지 마라

스킬 1,000개를 한번에 넣으면 로딩에 10분 이상 걸리고, 오히려 에이전트가 혼란을 겪는다. **필요한 것만 선택적으로 설치**하는 것이 핵심이다.

### 2. 충돌 확인

여러 프로젝트를 동시에 사용할 때, 같은 이름의 에이전트나 스킬이 충돌할 수 있다. 설치 전에 기존 설정과 겹치는 부분이 없는지 확인하자.

### 3. 커스터마이징 필수

어떤 프로젝트든 **그대로 쓰지 말고 내 상황에 맞게 조정**하자. 불필요한 에이전트는 제거하고, 필요한 규칙은 추가하는 것이 효과적이다.

### 4. 업데이트 주기 확인

오픈소스 프로젝트는 유지보수가 멈출 수 있다. GitHub의 최근 커밋 날짜, 이슈 응답 속도, 커뮤니티 활성도를 확인하고 선택하자.
